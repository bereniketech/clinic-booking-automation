## Objective
Write all Supabase PostgreSQL migrations, enable RLS with per-table policies, and add indexes. Every tenant-scoped table must have `clinic_id`. The `packages/db` package exposes a typed Supabase client used by both `apps/api` and `apps/workers`.

### Key Code Snippets

```sql
-- Pattern: every tenant table must follow this structure
CREATE TABLE <table_name> (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  clinic_id   UUID NOT NULL REFERENCES clinics(id) ON DELETE CASCADE,
  -- ... columns ...
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Pattern: RLS on every tenant table
ALTER TABLE <table_name> ENABLE ROW LEVEL SECURITY;
CREATE POLICY "<table_name>_clinic_isolation"
  ON <table_name>
  USING (clinic_id = (current_setting('app.clinic_id'))::uuid);

-- Pattern: advisory lock for double-booking prevention
SELECT pg_advisory_xact_lock(
  hashtext(staff_id::text || starts_at::text)
);
```

```typescript
// packages/db/src/client.ts — typed Supabase client pattern
import { createClient } from '@supabase/supabase-js'
import type { Database } from './types/database.types' // generated by supabase CLI

export const createDbClient = (supabaseUrl: string, serviceRoleKey: string) =>
  createClient<Database>(supabaseUrl, serviceRoleKey)

export type DbClient = ReturnType<typeof createDbClient>
```

### Key Patterns in Use
- **clinic_id everywhere:** `clinics` table has no `clinic_id`. Every other table does — no exceptions.
- **RLS as second defence:** Application layer enforces `clinic_id` first; RLS is the safety net.
- **Generated types:** Run `supabase gen types typescript` to produce `database.types.ts` — commit this file.
- **Migrations order:** `clinics` → `users` → `customers` → `services` → `staff_services` → `appointments` → `conversations` → `messages` → `workflows` → `workflow_runs` → `forms` → `form_responses` → `notification_schedules` → `audit_logs`

### Architecture Decisions Affecting This Task
- ADR-4: `clinic_id` enforcement at service layer — RLS is second line only
- All tables need indexes on `clinic_id` + the primary query field (e.g. `(clinic_id, phone)` on customers)
